# 单文件分步播放方案（full.mp4 + 时间轴 seek）

> 状态：**方案已定，待开发**  
> 目标：仓库与线上只保留一份 `full.mp4`，同时支持「完整播放」与「分步学习」；进度条展示剧情切分点。  
> 相关现状：`public/videos/{problemUuid}/{exampleUuid}/full.mp4` + `segments/*.mp4`（内容近似双份）；分步模式换源播小文件，未使用 manifest 时间轴。



> 此为未来计划！

---

## 1. 结论摘要


| 项     | 决定                                                                                                                            |
| ----- | ----------------------------------------------------------------------------------------------------------------------------- |
| 媒体文件  | **只保留 `full.mp4`**（+ `manifest.json` + `cover.png`）；**不再**产出/提交 `segments/`                                                   |
| 分步实现  | 同一 `<video>` 上按时间轴 **seek**，段末暂停，点「继续」进下一段                                                                                    |
| 段间空隙  | **保留** 渲染时的 `FULL_VIDEO_SEGMENT_GAP = 3s`（完整播放仍可见短暂停顿）                                                                        |
| 段边界定义 | 第 *i* 段播放区间 `**[startTime_i, startTime_{i+1})`**；最后一段 `**[startTime_n, duration]**`。即上一段的有效结束时间 = 下一段的 `startTime`（空隙归入上一段尾部） |
| 进度条   | 自定义（或覆盖）进度条，在各段 `**startTime**` 处打**切分标记**；悬停/点按显示 **名称（label）+ 时间（`0:19`）**；触屏在画面上**左右滑动**调进度                                |


---

## 2. 背景与问题

当前管线（见 `manim/scenes/_shared/post_render.py`、`export_segments.py`）：

1. Manim 渲染一条连续成片（段间 `wait(3)`）。
2. 复制为 `full.mp4`，并用 ffmpeg 按 `startTime`/`endTime` 切出 `segments/*.mp4`。
3. 前端「完整」播 `full.mp4`，「分步」播各分段文件。

问题：

- **存储近 2×**（full 与 segments 内容重叠；full 多含 3s 空隙）。
- 分步与完整是**同一条讲解的两种控制方式**，不应依赖两套文件。
- manifest 已有 `startTime`/`endTime`，前端未用于 seek。

---

## 3. 时间轴语义（核心）

### 3.1 录制侧（现状，可基本保留）

`SegmentRecorder` 在每段内容结束时写入 `endTime`（**不含**后续 3s gap），再 `wait(3)`，下一段 `startTime = 上一段 endTime + 3`。

示例：

```text
seg0: start=0.0   end=16.4   → wait 3s
seg1: start=19.4  end=35.0   → wait 3s
seg2: start=38.0  end=...
```

### 3.2 播放侧（本方案约定）

**有效播放区间不采用「录制 endTime」作为暂停点**，而采用：

```text
playbackEnd_i = startTime_{i+1}          // 非最后一段
playbackEnd_last = video.duration        // 最后一段
```

因此：

- 分步播放第 *i* 段时，会播完**内容 + 段末 3s 空隙**，再在下一段起点暂停（或等价地：播到 `startTime_{i+1}` 触发暂停）。
- 「上一段 end = 下一段 start」在**播放边界**上严格吻合，不再出现「切到 endTime 后还剩 3s 空白不知归谁」的歧义。
- 录制写入的 `endTime`（内容结束时刻）可保留为可选字段（见 §5），供调试、封面抽帧、统计「纯内容时长」；**分步控制逻辑以 `startTime` 链为主**。

### 3.3 与「去掉 3s」方案的区别

本方案**明确保留** 3s，完整观看仍有段间呼吸感；代价是完整模式进度条上两标记之间含空隙。若日后改为「空隙仅 UI、不进成片」，需另开方案（原讨论中的 A1）。

## 4. 产品行为

### 4.1 完整播放

- 单一 `full.mp4`，连续播放（含 3s 空隙）。
- 进度条上显示各段切分标记（§6）；点击标记可跳到该段 `startTime`（可选，建议做）。
- 原生 `controls` 若无法稳定画标记，则使用**自定义进度条**（推荐），或「原生 controls + 上方独立标记层」（命中区域要单独做）。

### 4.2 分步学习

- 进入分步：seek 到当前段 `startTime`，播放。
- 到达 `playbackEnd`（下一段 `startTime` 或片尾）：**暂停**，展示「继续」；不自动跳下一段。
- 「继续」/下一分段：seek 到下一段 `startTime` 并播放。
- 分段导航按钮：与现网一致，按 `label` 列表；点击即 seek 到对应 `startTime`。
- 双缓冲换源逻辑可废弃；预加载策略改为**单文件已在缓冲**（可按需 `preload=auto`）。

### 4.3 模式切换

- 完整 ↔ 分步：尽量保留 `currentTime`（落在哪段就高亮哪段）；从分步切完整时取消「等待继续」态即可。

## 5. 数据与产物

### 5.1 目录（目标态）

```text
public/videos/{problemUuid}/{exampleUuid}/
├── full.mp4
├── manifest.json
└── cover.png
```

不再包含 `segments/`。

### 5.2 manifest 建议字段

保持 `version` 可递增（例如 `3`），示意：

```json
{
  "version": 3,
  "problemId": "...",
  "exampleId": "...",
  "fullVideo": "full.mp4",
  "segmentGapSeconds": 3,
  "segments": [
    {
      "id": "...",
      "role": "intro",
      "label": "题型讲解",
      "startTime": 0.0,
      "endTime": 16.4,
      "hint": "可选"
    }
  ]
}
```

约定：


| 字段                  | 含义                               |
| ------------------- | -------------------------------- |
| `startTime`         | **切分点 / 段起点**（进度条标记位置）；必填        |
| `endTime`           | 内容结束时刻（gap 之前）；可选保留，**分步暂停不依赖它** |
| `file`              | 目标态可省略或忽略；兼容旧数据时可存在但不使用          |
| `segmentGapSeconds` | 文档化空隙；播放器可用常量 3 作兜底              |


**派生（播放器内计算，不必写入 JSON）：**

- `playbackEnd[i] = segments[i+1].startTime`（最后一段用 `duration`）
- 标记点列表 = 各段 `startTime` + `label`

### 5.3 管线改造（开发阶段再做）

- `post_render.py`：复制 `full.mp4` + 封面即可；**默认跳过** ffmpeg 切段（可留 `--split-segments` 仅供旧调试）。
- `export_segments.py`：标记废弃或仅工具用途。
- 文档 / Skill：更新「产出目录」说明。
- 已入库的 `segments/`：迁移脚本批量删除（开发期另开任务）。

## 6. 进度条切分标记（剧情点）

### 6.1 交互

- 在进度条轨道上，按 `startTime / duration` 换算为横向位置，绘制竖线或圆点（**切分标记**）。
- **时间格式（统一）**：`M:SS` / 超过 1 小时用 `H:MM:SS`，例如 `**0:19`**、`1:05`、`1:02:03`（分钟不强制两位；秒两位补零）。标记浮层、当前时间、总时长均用同一格式。
- **鼠标悬停**（桌面）：浮层显示  
  - 名称：`label`（如「题型讲解」「题目」「1」…）  
  - 时间：该点 `startTime`，格式同上（例：`题目 · 0:19`）
- **点击标记**：seek 到该 `startTime`（完整模式下开始播放或保持暂停策略与主进度条点击一致）。
- **触屏进度调节**：**在视频画面区域左右滑动**调节播放进度（类常见移动端播放器）  
  - 右滑：快进；左滑：快退  
  - 滑动过程中可显示临时浮层：当前目标时间（`0:19` 格式）± 偏移提示  
  - 与垂直滚动区分：以**水平位移为主**时才接管（避免与页面滚动冲突）；可用最小水平阈值（如 12～16px）再开始 seek  
  - 分步模式下滑动仍可在整段 `full.mp4` 时间轴上调节；若滑出当前段有效区间，按产品选择：允许跨段（推荐，与完整模式一致）或夹紧在当前 `[start, playbackEnd)`（实现时默认**允许跨段**）
- **触屏看标记信息**：无 hover —— **点按标记** → 短时展示 `label · 0:19` 并 seek（与桌面点击对齐）。

### 6.2 UI 示意

```text
|--●--------●--------●--------●--------●--------●--|
0  题型讲解  题目      1         作答      点拨      结尾
   ↑ hover / 点按: 「题目 · 0:19」

触屏: ← 左滑快退 | 画面区域 | 右滑快进 →
```

标记应：

- 不遮挡拖拽热区过多（可用细竖线 + 略大命中区）。
- 与当前播放段高亮联动（当前段两点之间轨道可用弱强调色）。
- 遵循现有中国风深色主题（低对比标记 + 悬停再提高对比）。

### 6.3 实现要点（前端）

- **不依赖**浏览器原生 chapters 轨道（兼容差）；自绘进度条或在 `video` 外包一层 slider。
- `timeupdate` / `seeked` 更新播放头；分步模式下到达 `playbackEnd - ε`（建议 ε ≈ 0.05～0.1s）触发暂停，避免浮点越过下一段。
- 完整模式与分步模式可共用同一套标记层；分步模式额外显示分段按钮条（现有 UI 可保留）。
- 时间格式化抽成共用工具（如 `formatVideoTime(seconds) => "0:19"`），进度文案与标记浮层共用。
- 触屏左右滑：在视频容器上监听 `pointerdown/move/up`（`pointerType === 'touch'` 或始终支持指针）；`touch-action: pan-y` 便于仍可纵向滚页面；水平手势内 `preventDefault` 避免跟手滚动打架。
- 滑动灵敏度：可按「水平像素 → 秒」换算（例如容器宽度对应整片时长，或固定 px/秒）；松手后 `currentTime` 落定并清浮层。

涉及文件（预期）：

- `src/composables/useSegmentedVideoPlayer.ts` → 改为 seek 模型（可重命名如 `useTimelineLessonPlayer`）
- `src/components/SolutionVideoPlayer.vue` → 自定义进度条 + 标记 + 触屏滑动
- `src/data/videoAssets.ts` → 探测逻辑改为「有 full + 合法 startTime 链即 ready」；不再要求 segments 文件存在
- `src/types/video.ts` → 状态与文案微调
- `src/utils/formatVideoTime.ts`（新建，可选）→ `0:19` 格式

## 8. 风险与注意点

1. **大文件拖拽**：单文件体积仍在；首屏可 `preload=metadata`，分步进入再抬高预加载。
2. **精确暂停**：必须在下一段 `startTime` 前停下，否则会漏播下一段片头；用 `timeupdate` + 容差，必要时 `requestVideoFrameCallback`。
3. **自定义进度条与无障碍**：需键盘可操作（左右键微调、Home/标记跳转可选）。
4. **Git 历史**：删 segments 不减小历史包体；若需减仓另议 LFS/历史重写（本方案不包含）。
5. **封面**：继续用现逻辑从 keypoints 等导出 `cover.png`，与是否切段无关。
6. **触屏滑动与点击冲突**：短触 vs 滑动需阈值区分，避免点按播放/标记时误 seek。

## 9. 可继续优化的点（方案层备忘，非本波必做）

在「单文件 + seek + 标记 + 左右滑」主干之外，下列点值得后续按优先级拣选。

### 9.1 体积与传输


| 点                 | 说明                                                 |
| ----------------- | -------------------------------------------------- |
| 编码参数统一            | 批量重导出时统一 CRF/preset、音频码率，避免部分课异常偏大                 |
| 分辨率档位             | 成片 1080p 保留；另可选导出 720p 供弱网（需双文件或自适应，接近非目标 HLS，慎开）  |
| Range / 缓存头       | Pages/CDN 确认支持 `Accept-Ranges`；前端大文件 seek 依赖 Range |
| 删除 segments 后的仓库  | 工作树体积下降；**git 历史仍胖**——长期用 LFS 或外置对象存储              |
| cover 与 keypoints | 已有 `cover.png`；确认不再为封面单独依赖整段 `keypoints.mp4` 播放    |


### 9.2 播放体验


| 点        | 说明                                                                  |
| -------- | ------------------------------------------------------------------- |
| 空隙体验     | 完整模式播到 3s 空镜时可显示弱提示「下一段：xxx」（可选，避免误以为卡死）                            |
| 分步自动续播开关 | 默认段末停；可加「自动进入下一段」偏好（本地存储）                                           |
| 倍速       | `0.75x / 1x / 1.25x / 1.5x`；分步暂停边界仍按真实 `currentTime`                |
| 记住进度     | 按 `problemId+exampleId` 记 `currentTime` / 段索引（刷新续看）；注意别与「未学完」进度业务混淆 |
| 键盘       | `Space` 播放暂停，`←/→` 微调，`,` / `.` 或数字键跳标记（可选）                         |
| 画中画 / 全屏 | 自定义控件时要接 `requestFullscreen`；PiP 可选                                 |
| 缓冲指示     | seek 到未缓冲区间显示 loading，避免「点了没反应」                                     |


### 9.3 分步与标记 UX


| 点       | 说明                                                       |
| ------- | -------------------------------------------------------- |
| 当前段高亮   | 进度条两标记间轨道着色 + 分段按钮 `aria-current`                        |
| 标记过密    | 段很多时合并命中或放大热区，防止点不准                                      |
| 滑动灵敏度   | 按片长自适应（短视频少滑多跳；长视频相反）；可配置                                |
| 双击左右半屏  | 常见 ±10s（与左右滑并存时要防误触，可仅桌面双击）                              |
| 完整/分步切换 | 保留 `currentTime`，并同步高亮段；分步切入若落在 gap 内则吸附到本段 `start` 或下一段 |


### 9.4 探测与健壮


| 点           | 说明                                                                     |
| ----------- | ---------------------------------------------------------------------- |
| 探针简化        | 目标态只需探测 `full.mp4` + 校验 manifest 时间轴，去掉对 N 个 segment 的 Range 探测（更快、更省） |
| manifest 校验 | `startTime` 严格递增；缺字段时降级「仅完整播放」并提示                                      |
| 片长与末段       | `loadedmetadata` 后用 `duration` 校正最后一段 `playbackEnd`；异常 duration 时兜底    |
| 旧资源清理       | 一次性脚本删 `segments/`，并写迁移说明避免本地缓存旧路径                                     |


### 9.5 管线与工程


| 点                     | 说明                                              |
| --------------------- | ----------------------------------------------- |
| `--skip-split` 变默认    | 与文档、Skill、`render_all_videos` 对齐                |
| manifest `version: 3` | 前端按 version 分支，减少双逻辑长期并存                        |
| 单测                    | `formatVideoTime`、`playbackEnd` 派生、时间轴单调校验纯函数单测 |
| 可访问性                  | 标记用 `role="button"` + 可读名称；进度条 `aria-valuenow`  |
| 分析（可选）                | 匿名记录「跳过哪些段」利于内容迭代（需隐私合规，默认关）                    |


### 9.6 明确仍不做（除非另开题）

- 成片去掉 3s（改渲染）
- HLS / 多码率 ABR
- 服务端动态切片
- git 历史重写减仓（需单独评估）

# Manim 制作流程

从编写脚本到前端可播放的完整流程（以第 1 讲为例）。

## 流程概览

```
编写 Scene → Manim 渲染 → finalize_lesson 写 manifest → post_render 归档 → 前端播放
```

## 1. 准备数据

确认 `public/data/problems/{lessonNumber}.json` 存在且 `methodType` 与脚本子目录一致。

题目文案变更若来自原始数据，需先跑：

```bash
node scripts/convertData.js
```

## 2. 编写 / 修改脚本

路径：`manim/scenes/{methodType}/problem_{lessonNumber}.py`

参考：`manim/scenes/图示法/problem_1.py`

## 3. 渲染

```bash
cd manim/scenes/图示法
```

| 命令 | 用途 |
|------|------|
| `python -m manim problem_1.py Problem1Scene -ql` | 低画质快速预览（480p15） |
| `python -m manim problem_1.py Problem1Scene -qh` | 高清成片（1080p60） |

临时输出（勿提交 git）：

```
manim/scenes/图示法/media/videos/problem_1/{quality}/Problem1Scene.mp4
```

渲染同时会执行 `finalize_lesson()`，将 `manifest.json` 写入 `public/videos/{problemUuid}/{exampleUuid}/`。

## 4. 后处理归档

```powershell
cd manim/scenes/图示法
python ..\_shared\post_render.py --lesson 1 `
  --rendered media\videos\problem_1\480p15\Problem1Scene.mp4
```

参数说明：

| 参数 | 含义 |
|------|------|
| `--lesson` | 讲次号，从 JSON 解析 problemUuid / exampleUuid |
| `--example-index` | 0=母题，1+=举一反三（默认 0） |
| `--rendered` | Manim 输出的完整 mp4 路径 |
| `--skip-split` | 仅复制 full.mp4，不切分段 |

产出：

```
public/videos/{problemUuid}/{exampleUuid}/
├── full.mp4              # 完整版（段间含 3 秒间隔）
├── manifest.json         # 分步导航（含 startTime/endTime）
└── segments/
    ├── cover.mp4
    ├── 01.mp4 … 07.mp4
    ├── keypoints.mp4
    └── end.mp4
```

## 5. 前端验证

```bash
npm run dev
```

打开第 1 讲解题页：

- **完整播放**：`full.mp4`
- **分步学习**：按 manifest 加载各 `segments/*.mp4`；切换模式应自动从「封面」段开始

## 6. 质量检查清单

- [ ] 段内标题与左上角标题无重叠
- [ ] 图解步骤与题目框、字幕无重叠
- [ ] 之间问题：中间未画满，待求区域标红完整
- [ ] 作答段：题目 + 列式 + 答案 + 缩小图同屏
- [ ] 点拨段：图解居中放大 + 点拨在下方
- [ ] manifest 段数、label 与代码 `segment()` 一致
- [ ] 分步切换能自动播放第一段

## 7. 举一反三视频

同一脚本改 `EXAMPLE_INDEX = 1`（或 2、3…），渲染后：

```powershell
python ..\_shared\post_render.py --lesson 1 --example-index 1 `
  --rendered media\videos\problem_1\480p15\Problem1Scene.mp4
```

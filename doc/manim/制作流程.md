# Manim 制作流程

从编写脚本到前端可播放的完整流程（前 9 讲已验证，第 10 讲起同流程）。

## 流程概览

```
核对 JSON 数据 → 规划（可选 tasks/）→ 编写/复用 diagrams → 编写 Scene
    → ql 预览 → 目视调整 → （可选）qh 成片 → post_render 归档 → 前端播放
```

**新讲次必读**：`.cursor/skills/math-drawing-video/references/new-lesson-workflow.md`

## 1. 准备数据

1. 确认 `public/data/problems/{lessonNumber}.json` 存在且 `methodType` 与脚本子目录一致
2. 与 `datas/draw_math_all_lessons.json` 母题精讲**一致**；勿混不同情境于同一母题
3. 题目文案变更若来自原始数据，需先跑：

```bash
node scripts/convertData.js
```

4. 查 [lessons-catalog.md](../../.cursor/skills/math-drawing-video/references/lessons-catalog.md) 是否已有可复用图解模块

## 2. 编写 / 修改脚本

路径：`manim/scenes/{methodType}/problem_{lessonNumber}.py`

| 讲次类型 | 参考 |
|----------|------|
| 通用结构（intro/question/written/keypoints） | `图示法/problem_1.py` |
| 线段图 + 进一法 | `线段图法/problem_9.py` |
| 复杂多区图解 | `图示法/problem_7.py` |

全新题型：先在 `manim/scenes/_shared/diagrams/` 新建 Mixin，并在 `lesson_base.py` 注册。

## 3. 渲染

```powershell
# 推荐：项目根目录单讲预览
python scripts/render_all_videos.py --quality ql --lessons 9
```

或进入脚本目录：

```powershell
cd manim/scenes/线段图法
python -m manim problem_9.py Problem9Scene -ql
```

| 命令 | 用途 |
|------|------|
| `-ql` | 低画质快速预览（480p15） |
| `-qh` | 高清成片（1080p60） |

临时输出（勿提交 git）：

```
manim/scenes/{methodType}/media/videos/problem_{N}/{quality}/Problem{N}Scene.mp4
```

渲染同时会执行 `finalize_lesson()`，将 `manifest.json` 写入 `public/videos/{problemUuid}/{exampleUuid}/`。

keypoints 段中，图解放大居中后会**自动导出大厅封面**（`export_hall_cover` → 视频目录 `cover.png`，与 `full.mp4` 同目录）。关闭：`EXPORT_HALL_COVER=0`。

## 4. 后处理归档

`render_all_videos.py` 会自动调用 `post_render.py`。手动时：

```powershell
cd manim/scenes/线段图法
python ..\_shared\post_render.py --lesson 9 `
  --rendered media\videos\problem_9\480p15\Problem9Scene.mp4
```

参数说明：

| 参数 | 含义 |
|------|------|
| `--lesson` | 讲次号，从 JSON 解析 problemUuid / exampleUuid |
| `--example-index` | 0=母题，1+=举一反三（默认 0） |
| `--rendered` | Manim 输出的完整 mp4 路径 |
| `--skip-split` | 仅复制 full.mp4，不切分段 |

产出：

```
public/videos/{problemUuid}/{exampleUuid}/
├── full.mp4              # 完整版（段间含 3 秒间隔）
├── cover.png             # 大厅封面（与 full.mp4 同目录）
├── manifest.json         # 分步导航（含 startTime/endTime）
└── segments/
    ├── 01.mp4 … 08.mp4   # 段数因图解步数而异
    ├── keypoints.mp4
    └── end.mp4
```

## 5. 前端验证

```bash
pnpm dev
```

打开对应讲解题页：

- **完整播放**：`full.mp4`
- **分步学习**：按 manifest 加载各 `segments/*.mp4`；切换模式应自动从「题型讲解」段开始

## 6. 质量检查清单

- [ ] 段内标题与左上角标题无重叠
- [ ] 图解步骤与题目框、字幕无重叠
- [ ] 分步图解与教材「画图分析」一致
- [ ] 辅助说明图全程保留（勿误 FadeOut）
- [ ] 多层 Brace 无文字重叠
- [ ] 作答段：列式前有文字说明（适用时）；题目 + 列式 + 答案 + 缩小图同屏
- [ ] 点拨段：图解居中放大 + 点拨在下方
- [ ] 大厅封面：视频目录 `cover.png` 存在且仅为图解（无「关键点拨」标题）
- [ ] manifest 段数、label 与代码 `segment()` 一致
- [ ] 分步切换能自动播放第一段

## 7. 批量渲染（一键全量）

见 [批量渲染.md](./批量渲染.md)（含自动封面说明）。

```powershell
python scripts/render_all_videos.py --quality ql    # 开发预览（含封面）
python scripts/render_all_videos.py --quality qh    # 正式成片（含封面）
python scripts/render_all_videos.py --quality ql --lessons 8,9
python scripts/render_all_videos.py --quality ql --lessons 1-60 --continue-on-error
```

## 8. 举一反三视频

同一脚本改 `EXAMPLE_INDEX = 1`（或 2、3…），渲染后：

```powershell
python ..\_shared\post_render.py --lesson 1 --example-index 1 `
  --rendered media\videos\problem_1\480p15\Problem1Scene.mp4
```

# 批量渲染所有视频

使用 `scripts/render_all_videos.py` 一键渲染并归档当前仓库内**全部** Manim 例题脚本。

## 扫描范围

自动发现：

```
manim/scenes/{methodType}/problem_{N}.py
```

当前覆盖第 **1～60** 讲。新增脚本后无需改配置，重新执行即可。

## 基本用法

在**项目根目录**执行：

```powershell
# 成片（1080p60，默认）
python scripts/render_all_videos.py

# 快速预览（480p15，开发调试用）
python scripts/render_all_videos.py --quality ql
```

每条讲次依次执行：

1. `python -m manim problem_N.py ProblemNScene -q{l|h}`
2. `python manim/scenes/_shared/post_render.py --lesson N --rendered ...`

## 产出路径

```
public/videos/{problemUuid}/{exampleUuid}/
├── full.mp4
├── cover.png                 # 大厅封面（与完整视频同目录）
├── manifest.json
└── segments/
```

### 大厅封面（自动）

渲染时**默认自动出封面**，无需第二套批处理：

| 项 | 说明 |
|----|------|
| 时机 | `play_keypoints_only`：图解放大居中后、点拨正文出现前 |
| 内容 | 按 `diagram` 包围盒裁切当前帧（仅图解，不含「关键点拨」标题） |
| 路径 | 与 `full.mp4` 同目录：`videos/{problemUuid}/{exampleUuid}/cover.png` |
| 实现 | `lesson_base.export_hall_cover` |
| 开关 | 环境变量 `EXPORT_HALL_COVER=0` 可关闭（默认开） |
| 画质 | 封面与视频画质无关；只补封面可用 `--quality ql` |

前端：`GridCard.vue` 读上述 `cover.png`，失败回退 methodType SVG。悬浮 tip 显示 `methodType` / `problemIdentification`。

## 常用参数

| 参数 | 说明 |
|------|------|
| `--quality ql` | 低画质快速预览（480p15） |
| `--quality qh` | 高清成片（1080p60，**默认**） |
| `--lessons 1,3,21` | 只渲染指定讲次（逗号分隔） |
| `--lessons 1-60` | 区间讲次（可与逗号混用，如 `1-10,21`） |
| `--example-index 0` | 0=母题，1+=举一反三（默认 0） |
| `--continue-on-error` | 某一讲失败时继续下一讲 |
| `--skip-render` | 跳过 manim，仅执行 post_render |
| `--skip-post` | 仅渲染，不归档 |
| `--dry-run` | 只打印命令，不执行 |

## 示例

```powershell
# 只重渲第 3 讲和第 21 讲（预览画质）
python scripts/render_all_videos.py --quality ql --lessons 3,21

# 全量预览（含封面），某一讲失败不中断
python scripts/render_all_videos.py --quality ql --lessons 1-60 --continue-on-error

# 全部成片；某一讲失败不中断
python scripts/render_all_videos.py --continue-on-error

# manim 已渲染过，只补跑归档（含已有 cover 同步）
python scripts/render_all_videos.py --skip-render --quality ql
```

## 前置条件

- Python 环境已安装 Manim CE（见 [环境搭建.md](./环境搭建.md)）
- `ffmpeg` 在 PATH 中（post_render 切分分段需要）
- Pillow 已安装（封面裁切写出 PNG；缺失则跳过封面并打印警告）
- 对应讲次的 `public/data/problems/{N}.json` 已存在

## 耗时说明

- `--quality ql`：每讲约 1～2 分钟，适合改动画后快速验收 / 补封面
- `--quality qh`：每讲约 5～15 分钟，适合发布前全量重渲

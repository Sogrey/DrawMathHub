# 小学数学画图解题法 - 项目规划

## 一、项目概述

**目标**：创建一个面向小学生的数学解题法学习平台，通过可视化的画图方式帮助学生理解和掌握各类数学题型的解题方法。

**核心功能**：
1. **目录展示** - 宫格形式展示60种解题方法，面向小学生，通俗易懂，可视化操作
2. **点击跳转** - 点击目录项（宫格），跳转到对应的解题页面
3. **规律分析** - 先展示这类题的规律分析切入口（找规律、找关系、找结构等），说明该使用哪种画图法解决比较合理
4. **例题引导** - 使用2-3道例题，一步步引导学生理解解题思路和解题过程，学生点击"下一步"才能继续
5. **练习区域** - 解题页面提供练习区域，学生在画板上练习该类题型解决方法，填写答案，系统判定是否正确并给出提示，答对了显示一个激励的动画再接再厉。
6. **动态生成** - 每次打开例题和练习题，重新生成新的题目和答案
7. **学习进度追踪** - 本地存储已学习的题型，在目录列表上显示标记，并记录练习题正确率

---

### 一.1 设计决策说明

**练习区域位置**：根据原始需求"解题页面需要提供一个练习区域"，练习区域设计在解题页面内部，采用标签页切换（学习/练习）的方式，而非独立页面。

**设计理由**：
1. **用户体验更优**：小学生用户无需在不同页面间导航，学习和练习可以无缝切换
2. **上下文保持**：练习时可以随时查看规律分析和推荐画图法作为参考
3. **操作更直观**：标签页切换比页面跳转更符合儿童认知习惯
4. **需求一致性**：完全符合原始需求文档的描述

## 二.1 画布引擎选型对比

经过调研，以下是几种主流二维画布渲染引擎的对比分析：

| 引擎 | 特点 | 大小 | Vue集成 | 适合场景 |
|------|------|------|---------|----------|
| **vue-konva (推荐)** | 声明式组件、图层管理、动画、交互事件 | ~50KB | 官方支持 | 交互式图形、数学绘图教学 |
| Fabric.js | 图形编辑、拖拽缩放、JSON导出 | ~70KB | 第三方封装 | 图形编辑器、白板 |
| Paper.js | 矢量路径、贝塞尔曲线、布尔运算 | ~100KB | 需自行集成 | 专业矢量绘图 |
| vue-drawing-canvas | 开箱即用、工具栏、撤销重做 | ~30KB | 原生Vue组件 | 简单绘图、签名 |
| draw-studio | 移动端支持、离屏Canvas优化 | ~20KB | Vue3组件 | 简单绘图板 |
| Two.js | SVG/Canvas/WebGL多渲染 | ~20KB | 需自行集成 | 动画效果 |

### 选型理由

**选择 vue-konva 的原因**：

1. **声明式语法**：与 Vue 3 的组合式 API 完美契合，使用组件化方式定义图形
2. **图层管理**：支持多层级渲染，方便实现复杂的绘图过程展示
3. **动画支持**：内置动画系统，适合解题步骤的逐步展示
4. **交互事件**：完善的事件处理机制，支持拖拽、点击等交互
5. **性能优秀**：基于 Konva.js 高性能渲染，适合频繁重绘场景
6. **社区活跃**：文档完善，问题响应及时

**适用场景划分**：

- **解题页面**（DrawingDisplay组件）：使用 vue-konva 实现画图过程的动画展示，逐步渲染解题步骤
- **练习页面**（Canvas组件）：使用 vue-konva 实现学生自由绘制，支持画笔、橡皮擦等工具

## 四、核心模块设计

### 4.1 数据模型

**Problem（题目）**
```typescript
interface Problem {
  id: number;
  title: string;
  method: string;           // 解题方法名称
  methodType: string;       // 方法类型：图示法、线段图法等
  description: string;      // 题型特点描述
  patternAnalysis: string;  // 规律分析切入口（找规律、找关系、找结构等）
  methodExplanation: string;// 推荐画图法说明（该使用哪种画图法解决比较合理）
  examples: Example[];      // 例题列表（2-3道）
  exercises: Exercise[];    // 练习题列表（每次动态生成）
}
```

**Example（例题）**
```typescript
interface Example {
  question: string;         // 题目描述
  answer: number | string;  // 答案
  steps: Step[];            // 解题步骤
  summary: string;          // 解题总结
}
```

**Step（步骤）**
```typescript
interface Step {
  instruction: string;      // 步骤说明
  drawingType: string;      // 绘图类型
  drawingData: any;         // 绘图数据
}
```

**Exercise（练习题）**
```typescript
interface Exercise {
  question: string;
  answer: number | string;
  blanks: Blank[];          // 填空位置
}

interface Blank {
  id: string;
  label: string;
  correctAnswer: number | string;
}
```

**LearningProgress（学习进度）**
```typescript
interface LearningProgress {
  problemId: number;        // 题目ID
  learned: boolean;         // 是否已学习（完成所有例题）
  practiceCount: number;    // 练习次数
  correctCount: number;     // 正确次数
  lastPracticeTime: number; // 最后练习时间戳
}

interface ProgressData {
  totalLearned: number;     // 已学习题型总数
  totalPracticed: number;   // 已练习次数
  totalCorrect: number;     // 已正确次数
  problems: LearningProgress[]; // 各题型学习进度
}
```

### 4.2 题目生成器

每个题型需要一个生成器函数，用于动态生成新题目。采用模板化策略：先实现18种方法类型的核心模板，再为60个题型提供参数化变体，避免重复代码。

```typescript
interface Generator {
  generateQuestion(): string;        // 生成题目描述
  generateAnswer(): number | string; // 生成答案
  generateSteps(): Step[];           // 生成解题步骤
  generatePatternAnalysis(): string; // 生成规律分析切入口
  generateMethodExplanation(): string; // 生成推荐画图法说明
  generateExamples(count: number): Example[]; // 生成指定数量的例题
  generateExercise(): Exercise;      // 生成一道练习题
}
```

**模板化策略**：
- 18种方法类型各对应一个核心模板（图示法、线段图法、列表法等）
- 每个模板包含该方法类型的通用生成逻辑
- 60个题型通过参数化配置调用模板生成题目
- 同一方法类型的不同题型通过调整参数（数字范围、场景描述等）实现差异化

## 六、关键功能实现要点

### 6.1 解题页面流程设计

### 1. 语音讲解功能

**方案选择**：使用 Web Speech API（浏览器原生 SpeechSynthesis），无需后端依赖，免费且即时响应。

**语音方案对比**：

| 方案 | 特点 | 优点 | 缺点 | 适合场景 |
|------|------|------|------|----------|
| **Web Speech API（推荐）** | 浏览器原生，零依赖 | 免费、即时响应、无需后端、隐私保护 | 语音质量依赖系统、部分浏览器兼容性差 | 教育类应用、辅助功能 |
| speak.js（eSpeak） | 纯前端 JavaScript | 多语言支持、可定制语音 | 文件体积大、语音质量一般 | 需要离线语音的场景 |
| 云 TTS（阿里云/腾讯云） | 云端 API | 语音质量高、支持情感语音 | 付费、需要网络、隐私风险 | 对语音质量要求极高的场景 |

**功能需求**：

1. **朗读内容**：
   - 题目描述朗读
   - 规律分析切入口朗读
   - 解题步骤讲解朗读
   - 答案解析朗读

2. **交互设计**：
   - 可朗读的文字后面添加喇叭图标
   - 点击喇叭图标开启语音阅读
   - 每次只朗读一条内容
   - 如果上一条正在阅读中，点击下一条时先停止上一条，再开始新的朗读

3. **播放控制**：
   - `speechSynthesis.cancel()` - 停止当前朗读并清空队列
   - `speechSynthesis.speak()` - 开始新的朗读
   - 监听 `onstart`/`onend` 事件更新喇叭图标状态

**实现方案**：

```typescript
// composables/useSpeechSynthesis.ts
export function useSpeechSynthesis() {
  const isSpeaking = ref(false);
  
  const speak = (text: string) => {
    if (!('speechSynthesis' in window)) return;
    
    window.speechSynthesis.cancel();
    
    const utterance = new SpeechSynthesisUtterance(text);
    utterance.lang = 'zh-CN';
    utterance.rate = 0.9;
    utterance.pitch = 1.0;
    
    utterance.onstart = () => { isSpeaking.value = true; };
    utterance.onend = () => { isSpeaking.value = false; };
    utterance.onerror = () => { isSpeaking.value = false; };
    
    window.speechSynthesis.speak(utterance);
  };
  
  const stop = () => {
    window.speechSynthesis.cancel();
    isSpeaking.value = false;
  };
  
  return { isSpeaking, speak, stop };
}
```

**UI 组件**：

```vue

<template>

    @click="toggleSpeech"
    :class="{ 'speaking': isSpeaking }"
  >
    <Volume2 v-if="!isSpeaking" :size="20" />
    <VolumeX v-else :size="20" />
  </button>
</template>
```

### 11.4 密码处理

**加密方案**：使用 Web Crypto API 的 SHA-256 哈希

```typescript
async function hashPassword(password: string): Promise<string> {
  const encoder = new TextEncoder();
  const data = encoder.encode(password);
  const hash = await crypto.subtle.digest('SHA-256', data);
  return Array.from(new Uint8Array(hash))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');
}
```

**验证流程**：
1. 用户输入密码
2. 对输入密码进行 SHA-256 哈希
3. 与数据库中存储的 passwordHash 进行比对
4. 如果匹配则登录成功

**安全性**：
- 从不存储明文密码
- 使用不可逆哈希算法（SHA-256）
- 密码验证在前端完成，无需后端
